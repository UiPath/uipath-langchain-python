# SQL Query Constraints (Calcite-based backend)

Write standard SQL. The following constraints define what does NOT work.

## UNSUPPORTED — Do NOT use these

### Patterns
- SELECT * — always use explicit column names
- COUNT(*) or COUNT(1) — use COUNT(column_name) instead
- Subqueries in WHERE or SELECT — use derived tables (subquery in FROM) instead
- Common Table Expressions (WITH/CTE)
- Window functions (ROW_NUMBER, RANK, PARTITION BY)
- Self-joins (same table joined to itself)
- FULL OUTER JOIN, CROSS JOIN
- UNION ALL — only UNION (deduplicating) works
- More than 4 tables in a JOIN chain
- HAVING without GROUP BY
- CASE WHEN in WHERE clause — only use CASE in SELECT and ORDER BY
- Arithmetic expressions in WHERE clause — only use arithmetic in SELECT
- NOT LIKE with patterns other than %value%

### Functions that do NOT work (use alternatives)
- COALESCE, IFNULL — use `CASE WHEN x IS NULL THEN default ELSE x END`
- SUBSTR — use `SUBSTRING(col, start, length)`
- LENGTH — use `CHAR_LENGTH(col)`
- LEFT(col, n), RIGHT(col, n) — use `SUBSTRING`
- LTRIM, RTRIM — use `TRIM`
- CONCAT(a, b) — use `a || b`
- Date functions (EXTRACT, DATE_ADD, YEAR, MONTH, DATEDIFF) — use LIKE on date strings (e.g., `WHERE date_col LIKE '2020%'`)
- POWER, SQRT, LN, LOG10, SIGN
- GROUP_CONCAT, LISTAGG, STRING_AGG
- CURRENT_DATE, CURRENT_TIMESTAMP
- EXISTS, NOT EXISTS, ANY, ALL
- IN with subquery
- JSON, REGEX, BLOB functions

### DML/DDL
- INSERT, UPDATE, DELETE, CREATE, ALTER, DROP — read-only access

## CRITICAL RULES

1. **ALWAYS use explicit column names** — never SELECT *
2. **Use COUNT(column_name)** — never COUNT(*)
3. **Maximum 4 tables** in a JOIN chain
4. **No subqueries in WHERE** — use derived tables in FROM instead
5. **No CTEs** — no WITH clauses
6. **No window functions**
7. **Explicit GROUP BY** — all non-aggregated columns in SELECT must be in GROUP BY
8. **Use SUBSTRING not SUBSTR**, CHAR_LENGTH not LENGTH, || not CONCAT()
9. **Use CASE WHEN for null handling** — COALESCE and IFNULL do not work
10. **For date filtering** use LIKE on string columns — date functions are not supported
